Discover memoQ 8.2

These are the most important new features of memoQ 8.2.

Productivity boosters

Enhanced X-translate

Whenever a source documents changes (imagine a car manual), memoQ has a feature called X-translation that allows to use the translation of previous versions wherever the source and target segments remained unchanged. When reimporting an updated source document, X-translation is now available for unchanged blocks of text that were moved within the document.

  1. Open the project that has the document you want to reimport.
  2. Select the document and on the ribbon, click Reimport.
  3. In the Document import options window, choose the filter and configuration and click OK.
  4. After the file is reimported, on the ribbon, click X-Translate.
  5. In the X-translate window, choose your settings and click OK.

    To learn more about reimporting: see the help page about Document import options.

    To learn more about X-translate: see the help page about X-translate.

 Multiple regex taggers

Create regex taggers before importing files and use more than one of them in cascading filters as you need. This way, you can import different files at the same time, using different cascading filters.

  1. Start importing a document (any monolingual format).
  2. In the Document import options window, select the documents, and click Change filter and configuration.
  3. The Document import settings window appears.
  4. Below Filter configuration, click the Add cascading filter link.
  5. The Select filter to chain window appears. From the Filter drop-down box, choose Regex tagger. Click OK.
  6. In the Document import settings window, the filter chain appears below the cascading filter controls. Click Regex tagger.

    To learn more about regex taggers: see the help page about regex taggers.

 Reimport multilingual Excel or XML files

Because it is a special format, until now, re-importing multilingual Excel files was not possible in memoQ. Now you can now re-import updated files and X-translate them. Number and formula handling also improved: you can exclude numbers from Excel files. It is now also possible to import the formula text and not the result.

  1. Open the project that has the document you want to reimport.
  2. Select the document and on the ribbon, click Reimport.
  3. In the Document import options window, choose the filter and configuration and click OK.
  4. After the file is reimported, on the ribbon, click X-Translate.
  5. In the X-translate window, choose your settings and click OK.

    To learn more about multilingual Excel files: see the help page about multilingual Excel files.

    To learn more about X-translate: see the help page about X-translate.

 Confirm segments without updating master TM

Until now, the Confirm and Update command confirmed the selected rows and immediately updated the Master TM. Now you can tell memoQ which TM to update - or to update no TM at all.

  1. Open the project that has the document you want to process.
  2. Select the document and on the ribbon's Statistics/Preparation tab, click Confirm And Update.
  3. In the Confirm and update rows window, choose your options and click OK.

    To learn more about confirming and updating: see the help page about Confirm and update rows.

 Easily import packages from other CAT tools

Import a package by simply right-clicking it and choosing the Translate with memoQ command. memoQ automatically reads the source and target languages from the files, and suggests a project name. You can create both local and online projects this way.

  1. Select a package file on your computer, right-click it and select the option Translate with memoQ.
  2. In the Publish project wizard (if you worked on a memoQ server) or Import package wizard (if you worked locally), choose your options.
  3. Change the type of project if needed:
    • To switch from online to local project: press Create project locally.
    • To switch from local to online project: press Create project on a server.
  4. Complete the wizard: press Next to step to the next page, and press Finish when you are done.

    To learn more about importing 3rd party packages: see the help pages about the Import SDL Trados Studio package, Import TIPP package, Import Transit package, and Import WorldServer package wizards.

 Categorize QA results into warnings and errors

In QA settings, you can choose which QA results should count as errors (they block delivery) or warnings.

  1. Open the Resource console and click QA settings.
  2. Select the QA settings profile you want to edit.
  3. Click the Edit link below the list.
  4. Click the Severity tab and make the changes you need.

    To learn more about QA severities: see the help page for Edit QA settings.

 memoQ WebSearch got faster and leaks no more

We replaced Internet Explorer with the Chromium engine in the popular WebSearch window. Not only does it solve the notorious memory leak with sites like Linguee, but it has become a lot faster too!

 Better tooltips

memoQ now features more explanatory tooltips to help you figure out new and existing features.

 Loading long project lists

When loading a long list of projects or TMs from a server, the user will see a “Loading” message and memoQ will not freeze.

Integrations

WordPress filter

WPML’s XLIFF files are somewhat peculiar, so until now, if you imported them normally, your text had a lot of HTML code and similar markup in memoQ. With the new, dedicated XLIFF filter, you can enjoy translating clean and properly segmented content – and get a preview, too.

  1. Open a local project. In Project home, choose Translations.
  2. In the Documents ribbon, click the down arrow below the Import icon.
  3. From the menu, choose Import with options.
  4. An Open window appears. Find and select the documents you want to import. Click Open.
  5. The Document import options window appears.
  6. Select the documents where you want to change the settings. At the bottom, click Change filter and configuration.
  7. In the Document import settings window, in the Filter dropdown, choose WordPress (WPML) filter.
  8. Choose an item from the Filter configuration dropdown and press OK.

    The document will be imported properly and you cna open it for translation.

    To learn more about the WordPress filter: see the help page for WPML XLIFF filter.

To learn more about importing documents: see the help page for Document import options.

 Rigi integration

Localize web applications with a high-quality in-context preview.

With Rigi, you can translate software strings in context. Open a project that supports Rigi. When you start translating, memoQ automatically opens a browser window showing the preview. Select a string in the translation editor and Rigi loads a preview with the existing translations.

To learn more about Rigi integration: see this page on Rigi's website.

Web Services API

memoQ 8.2 brings about additional functionality to the memoQ server Web Services API. There are new possibilities for existing integrations:

  • Document import into LiveDocs
  • Filtered listing of resources
  • Renaming resources and projects
  • Document import without preview
  • and more.

    To learn more about the Web Services API: see this page on the memoQ website.

Automation

Open workflows for vendors

Until now, when you assigned work to a vendor through online project packages, the vendor was limited to the 1, 2 or 3 workflow steps from the originating project. Vendors can now perform a full three-stage worfklow in a single step within the master project.

 Automated subvendor and GroupSourcing workflows

memoQ’s powerful project templates have so far only supported direct assignment and FirstAccept. We have now opened up the subvendor workflow and GroupSourcing for full automation.

  1. Open the Resource console.
  2. Create or find a suitable project template.
  3. Click Edit under the list.
  4. In the Edit project template window, click People on the left.
  5. Click the Languages and roles tab.
  6. Click one target language's row.
  7. In the Set users for target language window:
    • To use GroupSourcing: select the GroupSourcing radio button for the workflows that you want to handle this way.
    • To use subvendors: In the user list at the bottom, find the subvendor groups (created earlier on the server) that you want to work with. Check the boxes of the workflows you want to assign to them.

      To learn more about GroupSourcing: see the help page for Assign selected documents to users.

      To learn more about subvendors: see the help page for Assign documents to subvendor group.

 Manually export change track reports

The track changes review is a two-column report that includes source and target text as well as tracked-changes notations. Previously it could only be created using an automated action in a project template; we have now added a manual command to do the same.

Open an online project on a server and click Reports on the left.
Or:
Open a checked-out local copy of an online project. Click Overview on the left. Click the Reports tab at the top.

  1. Previously created reports are listed here, under Change tracked document. To create a HTML report from one, click Export and save the file.
  2. To create a new report, click Create new report now.
  3. In the Export change-tracked document window, choose your settings and click OK.
  4. The new report appears in the list. To create a HTML report from it, click Export and save the file.

    To learn more about change track reports: see the help page for Export change-tracked document.

 Sort and filter projects by creator

On the Dashboard, a new Created by column shows the user who created the project. You can sort and filter the project list using this column, for example to see only the projects you (or someone else) created.

To learn more about the Dashboard for translators: see the help page for Dashboard for translators.

To learn more about the Dashboard for project managers: see the help page for Dashboard for project managers.

Better access to QTerm TBs in server-to-server workflows

In some scenarios, QTerm’s rich metadata was lost when you created project packages for vendors because the offline term base stripped all custom fields. memoQ server now recognizes this situation and always includes a reference to the online QTerm term base instead.

 More efficient file export in Content Connected projects

In content connected projects, files can now be exported manually into the same folder structure to the one they were imported from.

  1. Open a content connected project.
  2. Select the files you want to export back.
  3. On the ribbon's Online Project tab, in the Document section, click the arrow under Content, and click Send Translation to Source.

    To learn more about content connected projects: see the help page about memoQ online project - Settings - Content source connection.

Machine translation

Centralized MT settings

Until now, each user had to configure machine translation (on collaborative projects) individually. With memoQ 8.2, project managers can configure MT for everyone in the project.

  1. Open the online project on the server.
  2. Click Settings on the left, then choose the MT settings icon from the top row.
  3. Select the configuration the translators will use and click the Edit link below.
  4. In the Edit machine translation settings window, make changes as needed and click OK.

    To learn more about MT settings: see the help page about Edit machine translation settings.

 Re-use MT configurations

Choosing a machine translation provider can depend largely on your language pair, domain and content type. You can now centralize and reuse these fine-tuned settings across similar projects.

  1. Open the Resource console and click MT settings.
  2. Click Create new below the list.
  3. In the Create new MT settings window, create a new MT profile.
  4. Select the new profile and click Edit below the list.
  5. In the Edit machine translation settings window, select and configure your provider(s) on the Services tab, then choose general MT options on the Settings tab.
  6. When you are done, click OK.

    To learn more about MT settings: see the help page about Edit machine translation settings.

 Formatting and inline tags in machine-translated segments

Microsoft and Google MT services in memoQ now support inline tags and formatting, saving you much tedious manual work to insert them in the right places.

  1. Open the Resource console and click MT settings.
  2. Select the profile you want to configure and click Edit below the list.
  3. In the Edit machine translation settings window, click the Google or Microsoft plugin on the Services tab.
  4. Choose an option from the Formatting and tags dropdown.
  5. When you are done, click OK.

    To learn more about MT settings: see the help page about Edit machine translation settings.

 MT in MatchPatch

Imagine you are translating a segment and your TM hit on the translation results pane only differs by just a few words, but these are missing from your term base. memoQ can now use the machine translation service to provide/correct the missing part.

  1. Open the Resource console and click MT settings.
  2. Select the profile you want to configure and click Edit below the list.
  3. In the Edit machine translation settings window, click the Settings tab.
  4. Choose your provider in the MatchPatch dropdown.
  5. When you are done, click OK.

    To learn more about MT settings: see the help page about Edit machine translation settings.

 Machine-translate selected text

Until now, machine translation worked only on full segments. Now, you can select text within a segment and have only that part machine translated.

  1. Open the Resource console and click MT settings.
  2. Click Create new below the list.
  3. In the Create new MT settings window, create a new MT profile.
  4. Select the new profile and click Edit below the list.
  5. In the Edit machine translation settings window, select and configure your provider(s) on the Services tab, then choose general MT options on the Settings tab.
  6. When you are done, click OK.
  7. Open the document in the translation editor.
  8. Select a part of a source segment and press Ctrl+Shift+K.

    To learn more about MT settings: see the help page about Edit machine translation settings.

    To learn more about MT concordance: see the help page about MT concordance.

To learn more about the translation editor: see the help page about the translation editor.

memoQweb

Client TMs on the web-based PM interface

Until now, if you received a TM from your client, together with the translatable files, memoQweb could only import it as a translatable document. Now you can import it as a TM.

  1. Log in to memoQweb and click Project management at the top.
  2. In the project management dashboard, click the Create new project link.
  3. Drag the files (or a ZIP file containing them) to the blue area, or click that area to browse for files.
 Relative paths in memoQweb

You can now see relative paths of files in the Translations view.

  1. Open an online project for management: From the Projects list, select the project you want to open. On the memoQ online project page, click the Translations icon on the left.
  2. In the Translations pane, check the Show paths of documents check box to see relative paths of files (path parts below the lowest common folder of all files) in the project.

Customer Portal

Customer Portal on memoQ cloud

Customer Portal is now available also for memoQ cloud.

To learn more about Customer Portal: read this page on the memoQ website.

Pricing

Previously, your customer could only receive word counts from the quotes they requested. Now, quotes will feature pricing information.

  1. Log in to memoQweb.
  2. On the Admin page, click Customer portal services.
  3. Find the service for which you want to set prices, and in the Ops column, click the Edit icon.
OR
  1. Create a new service: click New service at the top right.
  2. On the Create service or Edit service page, check the Prices check box.
  3. Enter or choose your currency in the Currency dropdown.
  4. For each language pair, enter the price in the Price (per weighted word) column.
  5. Click Save changes.

    To learn more about Customer Portal: read this page on the memoQ website.

 Cancelled and expired quotes

Whenever a quote is cancelled, on the Customer Portal it is moved to the Cancelled section, and in memoQ, the associated project is moved to the recycle bin.

Quotes can now also expire if they are not accepted within a set number of days. Such quotes also become cancelled.

Quote expiry:

  1. Log in to memoQweb.
  2. On the Admin page, click Customer portal admin.
  3. Under Expiry settings, set the length of the expiry period and the warning period.
  4. Click Save.

    Cancel a quote:

  5. Log in to memoQweb.
  6. On the Quote requests page, find the quote you need and in the Ops column, click the Open icon.
  7. On the Manage quote page, press Cancel quote.

    To learn more about Customer Portal: read this page on the memoQ website.

 Notifications

Customer Portal now features better notifications around the life cycle of translation jobs: project start and finish, completion of a workflow step, approaching or missed deadlines, etc.

If by any case the automation of quoting or translation project creation fails, the PM is notified automatically.

To learn more about Customer Portal: read this page on the memoQ website.

Manual QA step before delivery to client

Introduce an external step to your Customer Portal document before delivering the file to your customer (they all take place outside of memoQ). You can now perform QA, editing and testing on the documents once the translation workflow is finished.

  1. When translation and editing is finished in memoQ, export the translated files using the PM interface.
  2. Send them to external review, QA, testing or any other external step you have.
  3. When you receive the files back: select the project on the memoQ Dashboard, click the arrow under Deliver to Customer Portal, and click Deliver to Customer Portal with External Files.
OR
  1. Right-click the project on memoQweb's project management interface, and in the menu, click Deliver to Customer Portal with External Files.
  2. An Open window appears. Select the file you want to deliver and click Open.
  3. The file can be downloaded from Customer Portal, from the project's Details page in the Finished section.

    To learn more about Customer Portal: read this page on the memoQ website.

Security

Chat over HTTPS

When we initially introduced HTTPS for secure client-server communication, that option meant disabling the live chat in online projects. We have now removed this limitation: project participants can chat away again, even over HTTPS.

 Encrypted communication for Content Connectors

Communication with content connectors is now more secure. Channels can be encrypted, and we have introduced an API key with authentication for the management of the content connector. This way, you can install the memoQ server and Content Connector to different locations and they can communicate through a firewall.
